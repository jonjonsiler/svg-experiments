The following examples are using HTMLImageElement with a src attribute set to an external SVG file.

Uncompressed vs. Compressed

Compression largely reduces the vector data to it's minimum and does a more targeted styling at the element level rather than a SVG document level stylesheet.

Inline SVG text outline

We have an inline SVG+XML to test the utilization of fonts with the addition of an outline; which is an experimental CSS stage 3 standard. The SVG container does not have a fixed width (set at 100% of container).

Considerations:
  • We are able to load fonts into the base DOM to be used through standard styling techniques
  • Must be an inline SVG
  • ARIA-* attributes are available.
  • We have to correctly position the text SVG element using (x,y) coordinates; designs must be more composed and less responsive.
  • Centering gets tricky text-anchor: middle and percentage based positions
  • No word wrap
  • The output is clamped by the viewBox boundaries - width: 100% & height: 200px
